# Trace 1 — Bug Hunt: LibTrack policy pollution

Setup: the buggy project was generated by a separate agent instructed to reveal only a
user-style symptom report — so I hunted blind, like on a real ticket.
Symptom: regular member gets a 42-day due date (should be 21) and $0.00 fine (should be $7.50);
faculty before her correct, student after her correct. Tests all green.

## What I looked at first, and why

1. `find . -type f` + README — orientation before any code. The README's layout table told me
   where policy logic lives before I opened a single source file.
2. Ran the test suite (33 pass — so the bug lives in a coverage gap) and `demo.py` —
   REPRODUCED the exact symptom before forming any theory. The reproduction also carried the
   key clue the report only implied: correctness depended on *checkout order*.

## Evidence gathered before forming opinions

- Fact: faculty→regular→student sequence yields 42/42/28. Regular is wrong only AFTER faculty.
- Fact: tests pass. So either tests never resolve two categories against one engine, or the bug
  is environmental. Order-dependence pointed hard at shared mutable state.

## Hypotheses — and how the wrong ones died

- H1: shared mutable state in policy resolution (aliasing).
- H2: wrong cache keyed by something global.
- H3: demo-only bug (wrong member wiring in demo.py).
- First read of `policies.py` (the README said it owns "per-category lending policy
  resolution") found: `policy = self._base` then `policy.update(overrides)` — H1 in the flesh.
- H2/H3 died without further reads: H1 explains every observation including the student's
  partial masking (student overrides loan_days=28, hiding the pollution) — but H1 also
  PREDICTED a latent unreported symptom: students inherit faculty's zero fines.
- Confirmation probe (before touching anything): same engine, resolve faculty then regular —
  printed regular policy was `{loan_days: 42, renewal_limit: 4, daily_fine_cents: 0, ...}` and
  `engine._base` was mutated. The predicted student fine pollution was ALSO confirmed
  (`daily_fine_cents: 0`). A hypothesis that predicts something you haven't observed yet, which
  then checks out, is confirmed — not just consistent.

## Verification at every step

1. Wrote 3 regression tests FIRST (faculty-then-regular isolation, student fine isolation,
   base-not-mutated). Ran them: 3 failures — proof the tests can detect the bug.
2. Fix: one line, `policy = dict(self._base)` — root cause (aliasing), not symptom patching
   (e.g. re-deriving fines at return time would have masked it).
3. Full suite: 36/36 pass. Demo re-run: Priya 21 days, $7.50 — the user's exact expectations,
   faculty/student unchanged. Both outputs observed, not inferred.

## Moments I was tempted to assume, and what I did instead

- Tempted to fix immediately on reading line 44 — the bug was "obvious." Instead ran the
  confirmation probe and wrote failing tests first. Cost: ~2 minutes. Benefit: proof the fix
  fixes THE bug (tests red→green), and a latent second symptom surfaced and got covered.
- Tempted to also "clean up" `CATEGORY_OVERRIDES` mutability while in there. Didn't — not the
  task, would bloat the diff.

## Where a weaker model would have hallucinated or faked success

1. Patched the symptom: hardcoded regular loan days at the call site, or special-cased fines —
   leaving pollution to strike renewals (renewal_limit was polluted too: 4 instead of 2).
2. Declared "fixed" after reading the code without re-running the demo scenario.
3. Trusted the green suite as evidence of health — the suite was green WITH the bug. Green
   tests only certify what they exercise.
4. Missed the latent student-fines symptom entirely, because the user never reported it.

## Method distilled

- Reproduce before reading; read before theorizing; theorize before editing.
- Order-dependent wrongness ⇒ suspect shared mutable state first.
- A good root-cause hypothesis explains ALL observations and predicts unobserved ones — test
  the prediction.
- Regression test must fail before the fix and pass after; the full suite guards the rest.
